# 🏨 スマート客室リネン補充管理ツール (Linen Replenishment Calculator)

**「清掃現場の業務ロスと計算ミスをゼロにする、モバイルファーストなDXソリューション」**

ホテルの客室清掃（ハウスキーピング）現場における実用性を極限まで追求した、インストール不要で爆速動作するリネン計算・補充最適化ツールです。

---

## 🚀 プロジェクト概要

宿泊施設（ホテル、ゲストハウス、民泊など）のベッドメイキングや客室清掃時に使用するシーツやタオル等のリネン類について、**「本日の予定人数」**と**「前日の残数（現場のストック）」**を入力するだけで、以下の項目を1秒で自動計算するWebアプリケーションです。

1. **必要総数**（人数 × 1人あたりの使用量）
2. **倉庫から補充するべきセット（パック）数**（1セットあたりの枚数に合わせた切り上げ計算）
3. **翌日の余剰ストック数**（補充後の余り）

---

## 💡 開発背景と解決する課題（Why & Solution）

* **現場の泥臭い課題に着目：**  
  宿泊業界の慢性的な人手不足の中、客室清掃の「スピード」と「正確性」はホテルの稼働率に直結します。しかし多くの現場では、清掃スタッフが毎日「今日は〇人だからシーツは〇枚、タオルは〇枚...」と頭の中で計算し、さらに「1パック10枚入りだから、倉庫から何セット持ってくればいいか」を暗算で算出していました。
* **発生していた業務ロス：**
  - **計算ミスによる不足：** 清掃中にリネンが足りなくなり、倉庫を何往復もするロスタイムが発生。
  - **過剰補充による圧迫：** 必要以上に持ってきてしまい、リネン棚やカートから溢れて現場が乱雑になる。
  - **新人教育のコスト：** アイテムごとに「1人あたりの使用枚数」や「1パックあたりの枚数」が異なるため、新人スタッフが覚えるまでに時間がかかる。
* **ソリューション：**  
  「誰でも・スマートフォンから・瞬時に」正しい補充数をパック単位で把握できる仕組みを作ることで、**教育コストの削減、倉庫往復ロスの撲滅、適正ストックの維持**を実現しました。

## 🛠️ 技術スタック

あえて外部ライブラリ（ReactやjQueryなど）に依存せず、ブラウザの標準機能のみで動作する**Vanillaテクノロジー**で構築。現場の不安定なWi-Fi環境でも「一瞬で起動し、超軽量に動くこと」を最優先した設計です。

* **HTML5:** セマンティックなマークアップ、モバイルフレンドリーなビューポート設計、OGP（Open Graph Protocol）対応
* **CSS3:** カスタムプロパティ（CSS変数）、グラスモフィズム、キーフレームアニメーション、メディアクエリ（レスポンシブ）
* **JavaScript (ES6+):** DOM操作、動的要素生成、配列一元管理による拡張性、スムーズスクロール

## 💎 技術的なこだわり・アピールポイント (就活でのアピール事項)

就職活動において、単に「動くコードを書ける」だけでなく、**「実際のユーザー（現場のスタッフ）の目線に立った細やかなUX設計」**と**「美しく保守性の高いコーディング」**ができることをアピールするために、以下の点にこだわって実装しました。

### 1. 現場での片手操作を支える「超・ユーザー体験（UX）」
* **フォーカス時の全自動選択 (`this.select()`)**
  - 清掃用カートを押しながら、またはスマホを片手に持ちながらの入力操作を想定。数値入力欄（`input`）をタップした際、元から入っていた数値が自動的に全選択され、消去キーを押すことなく即座に新しい数値を上書き入力できるスマートな設計を採用しました。
* **モバイルファーストの「オートスクロール機能」**
  - スマートフォン（画面幅 768px 以下）での操作時、「計算する」ボタンを押すと、計算結果のテーブル部分へ画面がなめらかに自動スクロール（スムーズスクロール）します。ユーザーが手動で画面を下へスワイプする手間を解消しました。

### 2. ライブラリ不要の「レスポンシブ・テーブル・カード変換」独自実装
* スマホ画面で横長の表（Table）を表示すると、はみ出して横スクロールが必要になり視認性が著しく低下します。
* 本プロジェクトでは、CSSのメディアクエリと HTML の `data-label` 属性、およびCSSの `content: attr(data-label)` を駆使し、**「PC画面では表（Table）、スマホ画面ではカード形式のリスト」へ完全自動でレイアウト変換を行う仕組み**をピュアCSSで実装しました。これにより、デバイスを問わず最高の視認性を提供します。

### 3. 洗練されたプレミアム・グラフィックス
* **グラスモフィズム (Glassmorphism) の採用**
  - 背景にアニメーションを伴ってゆったりと浮遊するソフトカラーの球体を配置し、その前面に `backdrop-filter: blur(20px)` を適用した半透明のカードUIを重ねることで、清潔感と高級感のある最新のWebデザインを実現しました。
* **気持ち良いビジュアルフィードバック**
  - 計算ボタンをクリックした際、結果テーブルの各行が0.05秒ずつの時間差（ディレイ）を持って左から右へ心地よくフェードインするマイクロインタラクションをCSSアニメーションで実装。ユーザーに「使っていて気持ち良い」操作感を与えます。
  - 補充が必要なアイテム（持ってくるセット数が1以上）はピンクの背景で強調し、翌日の余りストックが2枚以下の不足リスクがあるアイテムは黄色（警告）で強調表示する「条件付き動的カラーリング」を実装しています。

### 4. 運用の変化に強い「保守性の高い疎結合設計」
* 将来的にホテル側で新しいリネンが導入されたり、取引先リネン業者の変更によって「1パックあたりのセット枚数」が変わった場合を想定。
* 計算ロジックをベタ書きせず、[main.js](file:///c:/Users/中山　皓貴/OneDrive/デスクトップ/アプリ開発/スマートプレスインC／Oセット作成/main.js) 内の `itemsConfig` 配列というオブジェクトモデルでリネンのマスタデータを一元管理しています。これにより、運用ルールが変わっても、数行の定義値を書き換えるだけでシステム全体が自動追従して動作します。

## 💡 計算ロジック

本アプリで採用している補充・最適化の計算式は以下の通りです。

1. **必要総数** ＝ `予定人数 × 1人あたりの使用枚数`
2. **追加補充必要枚数** ＝ `Math.max(0, 必要総数 － 前日残数)`  *(マイナスの場合は 0 とする)*
3. **持ってくるセット数** ＝ `Math.ceil(追加補充必要枚数 ÷ 1セットあたりの数量)` *(パック単位に切り上げ)*
4. **翌日余る数 (ストック)** ＝ `(持ってくるセット数 × 1セットあたりの数量) ＋ 前日残数 － 必要総数`
